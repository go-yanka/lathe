# `build-from-goal` — adversarial critique + hardened spec

Every hole below is a **real** mechanism in the current tree, cited to file:line. For each: the vacuity, then the deterministic code check that closes it. The proposed design is sound in intent but rests on primitives that today pass vacuously or don't exist.

## A. The spine is not actually on the path — routing holes

**A1 — `cmd_do` bypasses the entire spine.** `cmd_do` (lathe.py:168) calls `autonomy_live.run(goal, …, build_one=True)` directly. It never touches `cmd_flow`/`workflows.py`. It does **not** set `LATHE_STRICT` (verified: neither `cmd_do` nor `run_task` at autonomy_live.py:56-64 sets it), so today `lathe do` builds with STRICT *off* and no manifest. The design's "bare `lathe do` routes THROUGH its contract" is currently false at the source.
- **Close:** one `run_contract(invocation, goal, argv)` entrypoint mints the run-context and is the *only* caller of `autonomy_live.run`. Make `run()` refuse unless passed a `run_ctx` token minted in Phase 0 (a param the bare function cannot forge). Route `cmd_do`, `cmd_chat`, `cmd_auto`, **and** the MCP tool `mcp__lathe__lathe_do` through it. Add a static gate (`qa/run_gates.py`) that greps for any direct `autonomy_live.run(` / `live.run(` call outside `run_contract` and fails the tree — the bypass can't regrow.

**A2 — `you` steps are unrunnable no-ops that score PASS.** `classify_step('you', …)` returns `'todo'` **unconditionally** (flow_report.py:7), and `workflow_verdict` treats `'todo'` as PASS (flow_report.py:20-31; `[]`→PASS at :24). `cmd_flow` (lathe.py:930-933) *prints* a `you` step and appends `'todo'`. The design's steps 1, 7 (and half of 10) are `YOU→AUTO` "analyst answers, gated" — **there is no autonomous execution path for a `you` step**; in autonomous mode every judgment step is a printed no-op that passes. This is the single largest hole.
- **Close:** `build-from-goal` must contain **no `you` steps**. Every judgment step is an `auto` step backed by a real subcommand whose *output artifact* is gated by a following `gate` step (below). Delete reliance on `workflow_verdict`'s todo-is-PASS semantics: the manifest verdict is computed by a separate `manifest_verdict()` that is REFUSE unless every *required* phase has an explicit `ran`/`skipped-by-dial` record and every required gate an explicit `PASS`.

**A3 — `classify_step` is fed `""` as output, so only rc is checked.** `cmd_flow` calls `classify_step(kind, rc, "")` (lathe.py:944) — the substring failure-detector (`'traceback'`,`'error:'`… flow_report.py:5) never sees real output. A step that returns rc=0 having produced **nothing** (endpoint down, empty brief, empty ledger) is `pass`.
- **Close:** the runner must classify on the *produced artifact*, not on stdout heuristics: each auto step declares a `produces:` path + a `verify:` predicate (file exists, is fresh newer than run start, embeds this run's goal-hash, contains required sections). Missing/stale/foreign artifact → `blocked`, never `pass`.

## B. Per-step vacuous-pass holes

**B1 — clarify writes a "brief" from zero answers and returns 0.** `cmd_clarify` synthesizes the brief from `"\n\n".join(answers) or "(no answers provided)"` (lathe.py:1462) and writes `CLARIFIED_GOAL.md` whenever the endpoint returns non-empty, non-"API Error" text (lathe.py:1464-1489). Autonomous mode passes no answers → `questions:5, answered:0` yet the step PASSES. The manifest's `answered:5` is then a fiction.
- **Close:** autonomous clarify must run a second analyst pass that *answers each question*; the gate `goal-clarity` fails unless `answered == questions`. Record both counts from the actual Q/A objects, not a constant.

**B2 — `CLARIFIED_GOAL.md` is a single shared filename → stale-brief reuse.** clarify writes a fixed `CLARIFIED_GOAL.md` in `INNER` (lathe.py:1487). The design's "refuse if vague AND no CLARIFIED_GOAL exists" is satisfied by a brief left over from a *different* goal.
- **Close:** name it `plans/<slug>.CLARIFIED_GOAL.md` and embed `goal_sha256` in a front-matter line. The goal-clarity gate verifies the on-disk brief's embedded hash equals this run's clarified-goal hash; mismatch = no brief = refuse. Also require the brief to contain non-empty `Acceptance criteria:` and `Non-goals:` sections (structural check), else refuse — an endpoint that returns prose but no criteria is not a clarify.

**B3 — `goal_vagueness` is keyword-gameable in both directions.** It scores clear iff the string contains an `inputs`-keyword AND an `outputs`-keyword (clarify_logic.py:9-17). "return the result from the input" scores clear (contains `return`,`input`) though vacuous; a precise 5-word goal scores vague (`len<6`, :6). So the goal-clarity gate both false-passes and false-blocks.
- **Close:** keep the keyword score only as a *cheap trigger*, never as the pass condition. The pass condition is structural: goal-clarity PASSES only if (score-clear) **or** (a valid, hash-matched, section-complete brief exists per B2). The vague-and-no-brief branch REFUSES. A "clear" score never *skips* recording; it records `clarify: skipped-clear` so the evaluator sees the decision.

**B4 — no plan-file seam exists; author→build is atomic.** `autonomy_live.run` authors `plans/auto_*.py` on a **throwaway temp board+db** (`_goal_board()` lathe.py:159-165, removed in `finally`) and builds it in one opaque call. There is **no point** at which `assume {plan}`, `ack {plan}`, `lint-spec {plan}` can run on a materialized plan before build — the design's steps 3-9 have no seam to attach to, and the plan filename is `auto_*`, not `plans/<slug>.py`.
- **Close:** split into two subcommands: `author-plan {goal} → plans/<slug>.py` (persisted, not on a temp db) and `build {plan}`. The dispatcher binds `{plan}` from the author step's output and refuses to advance to any `{plan}`-scoped step if the file is absent/empty/not `is_valid_plan` (plan_validator). The `<slug>` derives from `goal_sha8` so it is stable and collision-checked.

**B5 — assumption gate: three vacuous-pass modes, all live.**
 (i) `LATHE_ASSUMPTION_POLICY=off` → advisory, never blocks (engine_v2.py:156). (ii) **Empty ledger passes** with only a printed advisory (engine_v2.py:177-179); a model self-audit returning "NO ASSUMPTIONS" collapses the ledger to `[]` (cmd_assume, lathe.py:1359), as does any endpoint output that `parse_assumptions` can't parse. (iii) `--accept-all` bulk-unblocks (lathe.py:1283-1299). The design's "HIGH-materiality blocks" is satisfied by an empty ledger, and "no blanket accept" is unenforced against an autonomous resolver.
- **Close:** (i) the dispatcher sets `LATHE_ASSUMPTION_POLICY` *authoritatively* from the dial and refuses if a weaker value is already in env (see B7). (ii) An **empty ledger is not a pass** under this contract: require a positive audit signal — the auditor must emit either ≥1 assumption *or* an explicit signed `NO_ASSUMPTIONS` token echoing the spec-digest; a bare-empty/parse-fail ledger → `assumption-audit: NOT-MEASURED` → REFUSE the strong verdict. (iii) In autonomous mode `--accept-all` is disabled; the resolver must decide each blocker with a recorded `via` ∈ {chose-alternative, stated-intent}; `accepted-as-is` on a HIGH blocker is recorded but flagged `self_resolved=true` because auditor and resolver are the same model — the manifest surfaces this so an evaluator never reads a self-rubber-stamp as independent review.

**B6 — test-ack is self-certifying.** `ack_ok` (engine_v2.py:125) just checks a digest in `.test_ack.json`. If the same automation that builds also writes the ack, the gate proves nothing.
- **Close:** the ack digest must be written by the *author-plan* actor and the build reads it; the manifest records `test-ack: auto (<model>)` — never presented as human review. The gate's real value in autonomous mode is only the *re-read-on-change* property (digest changes → re-ack), so enforce that the ack digest equals `tests_digest(FUNCTIONS)` at build time (engine_v2.py:126 already does) and record the actor.

**B7 — STRICT env forcing is order-dependent and defers to any preset env.** `strict_defaults` **skips any key already set** (strict_mode.py:24: `if current != '': continue`) and sets `LATHE_MUTATION_SCORE=0.5` (strict_mode.py:13) — which *contradicts* the dial's 0.7/0.8/0.9. So whoever writes the env first wins, and a hostile/careless parent env (`LATHE_MUTATION_SCORE=0.0`, `LATHE_ASSUMPTION_POLICY=off`, `SKIP_REGRESSION=1`) silently downgrades gates that STRICT will *not* restore. `run_task` propagates `dict(os.environ)` wholesale (autonomy_live.py:56) and clears only TRUST_PLAN/TRUST_REMOTE (:63-64), **not** SKIP_REGRESSION or the policy/threshold vars.
- **Close:** Phase 0 computes the dialed values and sets them **authoritatively** (overwrite, not setdefault), *after* clearing STRICT's own defaults for the keys the dial owns. Then assert a floor: refuse if effective `LATHE_MUTATION_SCORE` < dial floor, `LATHE_ASSUMPTION_POLICY` ∈ {off,advisory,''}, or `SKIP_REGRESSION`/`LATHE_TRUST_PLAN`/`LATHE_LINT_SPEC!=block` is set. Record the *effective* env table in the manifest. `strict_defaults` should switch from skip-if-set to a max()/min() reconciliation for numeric gates.

**B8 — several "PASS" gates are no-ops for greenfield.** (a) **regression-proof** only fires for a *changed* unit with an OLD impl (engine_v2.py:653) — a brand-new function has nothing to compare, so it's N/A yet the design lists it as a PASS gate. (b) **mutation-score is skipped for functions with no mutable AST nodes** — `mutation: unmeasurable … not gated` (engine_v2.py:702-703, banked in `_mutation_unmeasured`). A trivial `return x` mutation-passes vacuously. (c) **≥4 asserts is prompt text only** (autonomy_live.py:105/113/178) — *not* a gate; a 1-assert plan is not refused. (d) lint-spec's `spec_lint` only flags functions where a trivial impl is detectable; an un-analyzable function is "not weak" → passes.
- **Close:** the manifest must carry per-gate *measurement status*, not a bare verdict. `regression-proof` for new code → `N/A(new-unit)`, and lint-spec becomes the load-bearing stub gate (must be `block` + PASS). Add a code check: if *every* function in the plan is mutation-`unmeasurable`, the plan cannot claim `mutation-score: PASS` — record `NOT-MEASURED` and require adversarial-synth (real generated tests) to survive, else REFUSE the strong verdict. Enforce ≥4 asserts as an actual structural gate in the engine (`len(tests) >= LATHE_MIN_ASSERTS`, default 4) — refuse the plan otherwise, closing the "1 test" laundering.

**B9 — adversarial synthesis (step 10) does not exist, and pinning already happened.** There is no command that generates N new counterexample tests, runs them through `validate()`, folds survivors back and rebuilds — `review`/hreview is read-only CE persona prose. Worse, the engine **pins during the build** (step 8/9), *before* the design's step 10/11. So "refuse to pin if adversarial synthesis didn't run" is unenforceable — the pin exists before synthesis runs.
- **Close:** implement `adversarial-synth {plan}` as a real subcommand: analyst generates N tests (N from dial) → each is appended to a *copy* of the suite and run through the engine's `validate()` → any test the current impl fails is a surviving counterexample → folded into `FUNCTIONS[].tests`, which changes `tests_digest` and forces re-ack + rebuild. **Defer pinning:** the build step runs with `LATHE_DEFER_PIN=1` (new engine flag) so the module is gated-green but *unpinned*; pinning is a separate final step gated on `adversarial-synth ran AND produced ≥N cases AND 0 survivors`. No synth record → no pin → REFUSE.

**B10 — decider/fetch failures are laundered as "off-domain."** `select_agents_for_goal` is wrapped so any exception → `_picked=[]` (lathe.py:283) and `auto_spawn_for_goal` is best-effort → silently dropped (lathe.py:301-302). A selector crash or a *selected-but-unfetchable* security specialist reads in the manifest as "CE-floor only, nothing needed" — dangerous for auth/token goals.
- **Close:** distinguish three outcomes in code: `selected+available`, `selected+unavailable(fetch-failed)`, `selector-error`. Record raw match scores. If a persona was *selected* for a risk-bearing goal (security/data lens) and could not be fetched, that is a gate failure (or explicit REFUSE for security-class goals), never a silent proceed. Selector-error ≠ empty-selection.

## C. Manifest holes (skipped / incomplete / ambiguous)

**C1 — atexit is not a guarantee.** The design leans on an atexit/finally flush. But `sys.exit` on gate failure happens *inside the engine subprocess* (engine_v2.py:105,128,166,172) — the parent's atexit never sees it; and a hard `os._exit`/SIGKILL/OOM bypasses atexit entirely. "ALWAYS emitted" is violated on crash.
- **Close:** write the manifest **at intake, before any work**, with `verdict:"INCOMPLETE"` and every phase pre-seeded `status:"not-reached"`. After each phase, update-in-place as an append-only journal with `fsync`. atexit is only a *finalizer that flips INCOMPLETE→PASS/REFUSE and stamps `ended_at`*. A crash then leaves a present, parseable manifest whose last journal entry names the phase that was in flight. `ended_at`/`wall_ms` on crash = last journal timestamp.

**C2 — absence is ambiguous.** If a contributor/gate is missing from the arrays, an evaluator can't tell "skipped by dial" from "silently failed." 
- **Close:** the manifest schema **requires one record per spine phase and per required gate**, each with an explicit `status`. A `manifest_complete(m)` validator (pure, pinned) runs at finalize and forces `verdict:"INCOMPLETE"` if any required phase/gate record is missing or any required field family is absent. The *set* of required gates is computed by code from `(strict=1, plan_has_FUNCTIONS, greenfield)` — so an expected gate that produced no verdict (crashed) is a REFUSE, not an omission.

**C3 — cost family is unmeasured.** `request_spec` returns no token/usage data (verified: no `tokens`/`usage`/`cost` in request_spec.py). The manifest's `analyst_tokens_in/out`, `local_tokens`, `usd_est` are unfillable → fabricated or 0.
- **Close:** capture usage from the endpoint response where available; where not, record `null` + `"unmeasured"` — never 0. The validator flags the contradiction `model_calls.analyst>0 ∧ analyst_tokens_in==0` as INCOMPLETE, so cost can't silently read as free.

**C4 — pin-replay short-circuit can emit PASS for a broken tree.** "fully-pinned identical clarified goal → replay, 0 model calls, still emit M." Holes: clarify is itself a (possibly non-deterministic) model call, so hashing the *clarified* goal rarely hits; hashing the *raw* goal risks replaying a brief that was never produced. And a pin can exist while `tools/<module>.py` was hand-edited/deleted, or the surrounding tree regressed.
- **Close:** replay keys on `sha256(raw_goal)` *and* requires an existing hash-matched brief (else it's a fresh build, not a replay). Replay must **re-verify each pin's sha256 against the on-disk artifact bytes** and **still run standing regression** (`qa/run_gates.py`) — 0 model calls ≠ 0 gates. Any pin/file mismatch or regression → fall through to full build, never PASS-by-replay. Manifest records `pin_replay:true` with the verified sha per function.

**C5 — `SKIP_REGRESSION` honored in the do-path.** `run_task` propagates the full env (autonomy_live.py:56) and never clears `SKIP_REGRESSION`; engine skips standing regression if set (engine_v2.py:1063). A module can build green with no tree check.
- **Close:** Phase 0 clears `SKIP_REGRESSION` and refuses if the caller insists on it. Standing regression is a required gate (C2); TIMEOUT is already not-green (engine_v2.py:1119) — keep that, and record `standing-regression: TIMEOUT` as REFUSE, not absence.

## D. Hardened workflow (implementer's spec)

### D.0 Phase 0 — Intake (code, non-bypassable)
Single entrypoint `run_contract("build-from-goal", goal, argv)`; the only caller of `autonomy_live.run`, guarded by a static tree gate (A1). Actions: mint `run_id`; capture verbatim `argv`; resolve dial → **overwrite** env authoritatively (B7): `LATHE_STRICT=1`, `LATHE_TRIES`, `LATHE_MIN_ASSERTS=4`, `LATHE_MUTATION_SCORE=<dial floor>`, `LATHE_ASSUMPTION_POLICY=<dial>`, `LATHE_DEFER_PIN=1`, `LATHE_ADV_SYNTH_N=<dial>`; **clear** `SKIP_REGRESSION`, `LATHE_TRUST_PLAN`, `LATHE_TRUST_REMOTE_ANALYST`; **floor-assert** effective env, REFUSE if any gate weaker than dial. Write the INCOMPLETE manifest to disk with all phases `not-reached` and `fsync` (C1). Compute `raw_goal_sha`; attempt verified pin-replay (C4).

### D.2 Enforced guards (all √code)
1. No CLI→`autonomy_live.run` path outside `run_contract` (static tree gate). 2. No `{plan}`-scoped step advances without a materialized, valid `plans/<slug>.py`. 3. Empty assumption-ledger and empty mutation-measurement are `NOT-MEASURED`, never PASS. 4. No pin without a passing `adversarial-synth` record. 5. No `DONE` if any required gate lacks an explicit PASS. 6. Effective-env floor asserted at intake; weaker-than-dial → REFUSE. 7. Manifest written at intake, journaled+`fsync` per phase, finalized by validator — present on every exit including crash. 8. Verified pin-replay only (sha + standing regression), else full build.

### D.3 Manifest — required fields (validator-enforced)
Additions over the proposed schema, each **required-or-explicitly-unavailable**:
- `intake.effective_env`: the authoritative dialed values actually in force (mutation floor, assumption policy, tries, min_asserts, defer_pin, cleared vars) + `env_floor_ok:true`.
- `intake.goal_sha`, `intake.brief_sha`, `pin_replay:{replayed:bool, verified_pins:[{fn,sha,file_matches:bool}]}`.
- `phases[]`: **one record per spine phase**, `status ∈ {ran, skipped-by-dial, skipped-clear, failed, not-reached}` (absence impossible; validator forces INCOMPLETE otherwise).
- `selection`: raw match scores; each lens `availability ∈ {available, fetch-failed}`; `selector_status ∈ {ok, error}`.
- `contributors[]`: clarify carries real `questions`/`answered` (equal or REFUSE); assume carries `surfaced`, `blocking`, `resolved`, `self_resolved_high:int`; synth carries `cases_generated`, `survivors`.
- `gates[]`: **required set computed by code**; each has `verdict ∈ {PASS,FAIL,N/A,NOT-MEASURED}` + `measurement` (e.g. mutation `{score, per_fn_measured, per_fn_unmeasured}`); any required gate without a real verdict → REFUSE.
- `cost`: token/usd fields `null+"unmeasured"` where the endpoint gave no usage (never 0); validator flags `calls>0 ∧ tokens==0`.
- `verdict ∈ {PASS, REFUSE, INCOMPLETE}` — computed by `manifest_verdict()`/`manifest_complete()`, **not** `workflow_verdict` (whose todo-is-PASS/empty-is-PASS semantics are the vacuity engine, A2). `refusal:{phase,gate,reason}` on REFUSE; `incomplete_reason` on INCOMPLETE.

### Key code artifacts to build/patch
`strict_mode.strict_defaults` → max/min reconciliation, not skip-if-set (B7). New engine flag `LATHE_DEFER_PIN` (B9). New subcommands `author-plan`, `select`, `adversarial-synth` + `pin` split (B4,B9,B10). New pinned pure modules `manifest_verdict`/`manifest_complete` (C2). Engine `LATHE_MIN_ASSERTS` gate (B8c). Assumption gate: reject bare-empty ledger, add signed `NO_ASSUMPTIONS` token (B5ii). Static bypass gate in `qa/run_gates.py` (A1). Per-slug hash-embedded `CLARIFIED_GOAL.md` (B2).

The design's intent — deterministic spine, judgment-as-gated-skill, always-on manifest — is correct. But as specified it inherits four vacuity engines from the current tree: `you`-steps that score PASS, `classify_step` fed empty output, STRICT that defers to hostile env, and gates (assumption/mutation/regression-proof) that pass on *absence of measurement*. The hardening above converts every "did it run?" into a code-verifiable artifact check and every "PASS" into an explicit measurement, so nothing can be skipped, produce nothing, or pass vacuously without the manifest recording REFUSE/INCOMPLETE.
